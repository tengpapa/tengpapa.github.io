重装系统后，重新配置GitHub上的博客

发表于 2018-04-01 | 分类于 博客

重装系统以后，发现hexo deploy无法连上 GitHub 了。查了一下，原来本地存放的 GitHub 上配置的 SSH 的公钥和私钥已经不见了。下面就重新配置吧：

  1. hexo之前的配置在此就不讲了。
  2. 打开 git bash，运行 ssh-keygen -t rsa -C "youremail@example.com" 。运行完以后，会要求你输入密码，可输可不输，如果输入，hexo deploy的时候会要求你输入该密码，该密码的作用应该是将私钥加密进行本地存放。默认的目录下会生成.ssh目录，里面有 id_rsa 和 id_rsa.pub 两个文件。
  3. 登陆 GitHub，打开「Settings」->「SSH and GPG keys」，然后点击「new SSH key」，填上任意 Title，在 Key 文本框里粘贴公钥 id_rsa.pub 文件的内容，最后点击「Add SSH Key」，就可以看到已经添加的 Key。至此，在 GitHub 网站上的设置已经完成了。
  4. 运行git config --global user.email "you@example.com"，git config --global user.name "Your Name"。
  5. 然后就可以发布网站了。
