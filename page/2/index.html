C语言的历史

发表于 2017-03-16 | 分类于 C

　　笔者在研究一种事物时，喜欢了解它的历史背景，这样有助于对该事物进行更全面的认识，尤其是当你喜欢一种事物时，你更需要去了解它的前世今生。

　　可能大家都知道 C 语言是在 B 语言的基础上发展来的，但可不是因为是排在 B 后面所以就起名叫 C 语言的。追溯 C 的根源，需要从 ALGOL60 讲起。

C 的前世及诞生

　　1960 年，ALGOF60 诞生，它是一种面向问题的高级语言，它是程序设计语言由技艺转向科学的重要标志，其特点是局部性、动态性、递归性和严谨性，但它离硬件比较远，不宜用来编写系统程序。

　　1963 年，英国剑桥大学推出了 CPL 语言。CPL 语言在 ALGOL60 的基础上接近硬件一些，但是规模比较大，难以实现。

　　1967 年，英国剑桥大学的 Martin Richards 对 CPL 语言做了简化，推出了 BCPL 语言。

　　1970 年，美国贝尔实验室的 Ken Thompson 以 BCPL 语言为基础，又进一步简化，设计出了很简单的而且很接近硬件的 B 语言（取 BCPL 的第一个字母）。

　　1971 年，在 PDP 11/20 上实现了 B 语言，并编写了 UNIX 操作系统，此时的 B 语言过于简单，功能有限。

　　1972 年至 1973 年间，贝尔实验室的 Dennis M.Ritchie 在 B 语言的基础上设计出了 C 语言（取 BCPL 的第二个字母）。 C 语言既保持了 BCPL 和 B 语言的优点（精炼，接近硬件），又克服了它们的缺点（过于简单，数据无类型等）。最初的 C 语言只是为了描述和实现 UNIX 操作系统提供一种工作语言而设计的。

C 的早期发展

　　1973 年，Ken Thompson 和 D.M.Ritchie 合作把 UNIX 的 90% 以上用 C 语言改写（即 UNIX 的第 5 版。原来的 UNIX 操作系统是 1969 年由美国的贝尔实验室的 Ken Thompson 和 D.M.Ritchie 开发成功的，是用汇编语言编写的）。

　　1975 年 UNIX 第 6 版发布后，C 语言的冲突优点才引起人们的普遍注意。

　　1977 年出现了不依赖于具体机器的 C 语言编译文本《可移植 C 语言编译程序》，使 C 语言移植到其他机器时所需做的工作大大简化了，这也推动了 UNIX 操作系统迅速地在各种机器上实现。例如 VAX、AT&T 等计算机系统都相继开发了 UNIX。随着 UNIX 的日益广泛使用，C 语言也迅速得到了推广。

　　1978 年以后，C 语言先后移植到大、中、小、微型计算机上，已独立于 UNIX 和 PDP 了。以 1978 年发布的 UNIX 第七版中的 C 语言编译程序为基础，Brain W.Kernighhan 和 Dennis M.Ritchie（合称 K&R）合著了影响深远的名著《The C Programming Language》（中文译名为《C程序设计语言》），这本书中介绍的C语言成为了后来广泛使用的 C 语言版本的基础，它成为事实上的 C 标准。人们称这个版本的 C 语言为“K&R C”。

　　1988 年，Brain W.Kernighhan 和 Dennis M.Ritchie 修改此书，出版了《The C Programming Language》第二版，第二版涵盖了 ANSI C 语言标准。第二版从此成为大学计算机教育有关 C 语言的经典教材，多年后也没再出现过更好的版本。

C 的标准化

　　1970 到 80 年代，C 语言被广泛应用，从大型主机到小型微机，也衍生了 C 语言的很多不同版本。

　　1983 年，美国国家标准协会（ANSI）根据 C 语言问世以来各种版本对 C 语言进行了发展和扩充，公布了第一个C语言标准草案（83 ANSI C）。ANSI C 比原来的 C 有了很大的发展。

　　K&R 在 1988 年修改了他们的经典著作《The C Programming Language》，按照即将公布的 ANSI C 重新写了该书。

　　1989 年，ANSI 公布了一个完整的 C 语言标准－X3.159.1989，简称 C89，不过人们也习惯称其为“ANSI C”。

　　1990 年，国际化标准组织 iOS（International Standard Organization）接受 C89 为 IOS C 的标准，通称 C90。C90 与 C89 基本相同，主要是格式组织不一样。

　　注意：以后大家看到 ANSI C、ISO C、C89、C90，要知道这些标准的内容都是一样的。目前，几乎所有的开发工具都支持 ANSI / ISO C 标准，是 C 语言用得最广泛的一个标准版本。

　　在 ANSI C 标准确立之后，C语言的规范在很长一段时间内都没有大的变动。1995 年 C 程序设计语言工作组对 C 语言进行了一些修改，成为后来的 1999 年发布的 ISO/IEC 9899:1999 标准，通常被成为 C99。

　　但是各个公司对 C99 的支持所表现出来的兴趣不同。，这就导致了有的编译器并未实现 C99 所建议的所有功能。

　　在 2011 年 12 月 8 日，ISO 又正式发布了新的标准，称为 ISO/IEC9899: 2011，简称为“C11”。

总结

　　其实写这么多，就是为了把 C 的历史背景交代清楚。可能有的人会觉得啰里啰嗦，不过没关系，其实你只需要知道几个重要的知识点就 ok 啦！

  • C 语言是在1972年被一个叫 Dennis M.Ritchie（C语言之父）的人设计出来的。
  • C 语言是在B语言的基础上设计的，是为编写 UNIX 操作系统而被发明的。
  • C 的正式标准是 1989 年 ANSI 发布的。
  • 现行的 C 标准有 2 个：C89 和 C99。但有的编译器不完全支持 C99。后来又来了个“C11”，支持情况不明。

写在前面的话

发表于 2017-03-14 | 分类于 感悟

“为什么要写技术博客？”，一直以来没有认真思考过这个问题。之前断断续续写过一些，主要是在CSDN上。可是后来发现CSDN的广告已经严重地影响了我的正常浏览，于是决定放弃。新的博客平台选择哪里呢？在网上开始撒网查找。无意间发现了一个作者写的《为什么你要写博客？》一文。作者的一番话，让我重新认识了写博客的意义。我突然觉得之前写的博客，太简单，太功利，纯粹就是记笔记性质的，没有做过梳理，没有做过总结，甚至写完自己几乎从没看过。想来也是惭愧，看来自己对写技术博客还是没有好好地认识。结合作者写的，我重新分析了一下自己写博客的目的：

  • 积累更多知识。这个很好理解，我之前可能只认识到了这一点。
  • 提高将事情讲清楚的能力。而这恰恰是我所欠缺的，在讲问题的过程中可能又会发现新的问题，甚至有的地方自己都觉得讲不过去，那这个时候就需要你再进一步地进行学习与理解，进一步进行总结与概括。
  • 培养持续做一件事的能力。默默地持续做一件事是一种难得的能力，也是一种难得的品质。我想如果我能坚持，N久后，回过头再翻看前面的博客，也许会觉得之前的认识有些可笑，也许会觉得之前的想法有些超前，但无论如何相信肯定会有满满的成就感。

所以我要重新认识写博客，开始坚持写博客，让写博客成为一种习惯。当然，我还得时刻提醒自己，一定要注重质量。博文贵在精，而不在多。加油！
